Android笔记: 基础知识

发表于 2017-10-05 | 分类于 技术

Android 主要采用Java语言编写，Android Sdk将代码数据和资源打包编译到一个apk文件中。这个文件也就是 Android 系统的安装文件。程序安装后，每个应用程序都运行在自己的安全沙箱内，以保证应用实现最小权限原则，相互隔离，提供一个安全的运行环境。

应用组件

Activity:提供用户界面相关
Service:运行于后台，不提供界面。用于长时间运行的组件
ContentProvider:为存储和获取数据提供统一的接口,实现在应用程序之间共享数据
BroadcastReceiver:

启动组件

Activity、Service、BroadcastReceiver 通过 Intent 的异步消息启动
Intent 不会直接启动 ContentProvider组件，通过成为 ContentResolver 的请求目标时启动，在 ContentResolver 对象上调用方法，使请求与组件间留出抽象层，确保安全。
Intent 可以是显示的，在清单文件中，通过组件类名执行操作。
也可以是隐式的，在清单文件用<intent-filter>标签过滤器类型匹配执行。

  • Activity 通过 startActivity() 或 startActivityForResult() 传递 Intent
  • Service通过startService()启动bindService()传递Intent
  • BroadcastReceiver 通过 sendBroadcast()、sendOrderedBroadcast() 或 sendStickyBroadcast() 传递 Intent
  • ContentProvider 通过 ContentResolver 上调用 query() 查询

清单文件

每个应用根目录必须包含AndroidManifest.xml文件，系统启动应用前，会读取AndroidManifest.xml文件确认组件存在.
作用:

  • 为软件包命名，唯一标识符
  • 声明组件、以及启动条件信息
  • 确定组件的进程
  • 确定权限
  • 声明最低API级别
  • 声明使用软硬件功能
  • 连接其他API库
  • 列出 Instrumentation 类，应用运行时提供分析和其他信息，发布前移除

注意：

  • 当启动组件时，Android 系统会创建已命名子类的实例。如果未指定子类，则会创建基类的实例
  • </manifest> 结束标记必须紧接在 </application> 结束标记后
  • 属性值为字符串，必须使用双反斜杠 (\\) 转义字符，例如，使用 \\n 表示换行符或使用 \\uxxxx 表示 Unicode 字符。
  • 如果应用需要访问受权限保护的功能，使用 <uses-permission> 声明需要该权限。应用安装到设备上后，安装程序会通过
    检查签署应用证书的颁发机构并（在某些情况下）询问用户，确定是否授予请求的权限。如果授予权限，则应用能够使用受保护的功能。
    否则，其访问这些功能的尝试将会失败，并且不会向用户发送任何通知。

组件声明

基于Android系统的设备多样性，并非所有设备具有相同的特性和功能，为防止应用安装在缺少所需特性的设备上，必须通过AndroidMainfest文件中声明。
系统不会读取这些声明，大多声明只是提供信息，为外部商店服务读取，为用户搜索应用时提供过滤功能。

没有相机且 Android 版本低于 2.1 的设备将无法从商店安装应用，也可以将 required 属性设为 “false”，并在运行时检查设备是否具有相机，根据需要停用任何相机功能。
