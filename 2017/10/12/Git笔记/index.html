Git笔记

发表于 2017-10-12 | 分类于 技术

开发者的必备技能，总结记录一些常见的操作和命令，仅供自己备忘。也欢迎和大家交流。

基本概念：

Git是一个开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。
git四大域：工作目录，暂存区，仓库，远程仓库。
版本库又名仓库，英文名repository，你可以简单理解成一个目录，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以“还原”。

origin就是一个名字，它是在你clone一个托管在Github上代码库时，git为你默认创建的指向这个远程代码库的标签
git的服务器端(remote)端包含多个repository，每个repository可以理解为一个项目。而每个repository下有多个branch，”origin”就是指向某一个repository的指针。服务器端的”master”（强调服务器端是因为本地端也有master）就是指向某个repository的一个branch的指针。
而在本地电脑(local)上：”master”就是指向刚刚从remote server传到本地的副本branch。

本地操作

系统环境为mac,所有命令在终端命令行内执行。$为命令行输入提示符。

创建本地版本库

打开终端窗口，在已有的目录或新建一个目录里,命令行执行执行git init

通过此命令把目录变成 Git 可以管理的仓库，目录下多了一个 .git 的目录，这个目录是Git来跟踪管理版本库的。默认为隐藏文件 ls -a 可查看

添加到暂存区

在初始化好的仓库目录下，新建一个文本文件 hello.txt 内容如：

Hello World

然后命令行执行 git add hello.txt添加文件到暂存区。

查看仓库当前状态

执行 git status 命令后，可查看到当前仓库的状态，hello.txt 已加入暂存区,等待提交

一般有三种状态:

  • Untracked files：未被跟踪的文件，表示是工作目录新增加的文件
  • Changes not staged for commit：工作目录中修改了文件，但是没有被添加到暂存区
  • Changes to be committed:添加到暂存区的文件，等待提交

提交到本地仓库

此时 hello.txt 已经添加到暂存区，可用git commit 命令把暂存区的修改提交到本地仓库中保存。

查看提交日志

随时可用 git log 命令查看提交日志

commit 显示的一串字符是本次提交的 commit id 可用来回退版本

回退版本

再次修改 hello.txt 文件,内容如：

Hello World
你好 世界

修改后，添加到暂存区并提交
