Java笔记：初始化数据顺序引发的BUG

发表于 2017-10-23 | 分类于 技术

在处理串口回复据的解析时，因为回复数据格式相同，只是内容消息体不同，所以想通过统一封装回复响应数据，做一些公共处理，然后由不同子类各自处理消息体内容。踩了一个初始化顺序的坑，特此记录总结。

这个象的赋值顺序。

  1. 进入 Sub 构造函数。
  2. Sub 成员变量的内存被分配。
  3. Base 构造函数被调用。
  4. Base 构造函数调用 Sub 的重写 init()，设置 name = “hello” 。
  5. Sub 的成员变量初始化被调用 name = “”
  6. 执行 Sub 构造函数体。

关键在与 String name = “” 这句，赋值的成员变量，声明和赋值只是看起来是写成了一句，但并不是立即赋值，而是在执行了父类构造函数之后执行。
