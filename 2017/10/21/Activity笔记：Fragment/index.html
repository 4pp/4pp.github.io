Activity笔记：Fragment

发表于 2017-10-21 | 分类于 技术

Fragment 用来处理 Activity 中的行为或用户界面部分。可以将多个 Fragment 组合在一个 Activity 中来构 UI，以及在多个 Activity 中重复使用某个 Fragment。可以将 Fragment 视为 Activity 的模块化组成部分，它具有自己的生命周期，能接收自己的输入事件，并可以在 Activity 运行时添加或移除 Fragment。Fragment 必须始终嵌入在 Activity 中，其生命周期直接受宿主 Activity 生命周期的影响。

将 Fragment 作为 Activity 布局的一部分添加时，它存在于 Activity 视图层次结构的某个 ViewGroup 内部，并且 Fragment 会定义其自己的视图布局。通过在 Activity 的布局文件中声明片段，将其作为 <fragment> 元素插入 Activity 布局中，或者通过将其添加到某个现有 ViewGroup，利用应用代码进行插入。还可以将没有自己 UI 的 Fragment 用作 Activity 的不可见工作线程。

通过上面的官方描述，做出如下总结。

Fragment 特性

  • 必须放在 Activity 中使用
  • 具有独立的生命周期，但会受宿主 Activity 生命周期的影响
  • 具有独立的任务栈处理导航
  • 宿主 Activity 可对 Fragment 处理增删改查管理

Fragment 作用

  • 可使界面代码功能模块化，独立封装。
  • 更容易被多个 Activity 复用。
  • 可以更灵活的UI设计，适应于不同的屏幕尺寸。从手机到平板电脑。

生命周期

因为 Fragment 是依附于 Activity 存在的，因此它的生命周期收到 Activity 的生命周期影响

Fragment 比 Activity 多了几个生命周期的回调方法

  • onAttach(Activity) 当 Fragment 与 Activity 发生关联的时候调用
  • onCreateView(LayoutInflater, ViewGroup, Bundle) 创建该 Fragment 的视图
  • onActivityCreated(Bundle) 当 Activity 的 onCreated 方法返回时调用
  • onDestroyView() 与 onCreateView 方法相对应，当该 Fragment 的视图被移除时调用
  • onDetach() 与 onAttach 方法相对应，当 Fragment 与 Activity 取消关联时调用

注：除了 onCreateView ，其他的所有方法如果重写了，必须调用父类对于该方法的实现

使用总结：

  1. 将每个 Fragment 都设计为可重复使用的模块化 Activity 组件。
  2. 避免直接从某个片段直接操纵另一个 Fragment
  3. 尽量不要用app包中的fragment，因为这个是在3.0之后才有的，支持的版本太高，在低版本中是是用不了的,android.support.v4.app.Fragment：可以兼容到1.6的版本。
  4. 当使用 v4.fragment 时, Activity 的布局中使用这个 Fragment 的 <fragment> 标签的时候，这个 Activity 必须继承 FragmentActivity，否则会报错。
