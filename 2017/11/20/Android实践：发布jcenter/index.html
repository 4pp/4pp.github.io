Android实践：发布jcenter

发表于 2017-11-20 | 分类于 技术

采用 bintray-release 插件快速发布项目到jcenter

环境

java

jdk 1.8.0_151

gradle-wrapper.properties

distributionUrl=https\://services.gradle.org/distributions/gradle-4.3-all.zip

build.graple

classpath 'com.android.tools.build:gradle:2.3.0'

准备

项目中的libary 是一个 Android library 模块。发布到jcenter中

注册

1.https://bintray.com/ 注册账号，然后激活登录。
2.登录后 Add New Repository 新建仓库

填写示例

配置

1.在项目的 build.gralde 文件中配置

注：注意是项目的 build.gradle，不是module的
2.上传 module 的 build.gralde 文件中配置

上传

查看Api Key

复制这个key

执行上传命令，在 Android Studio 的 Terminal 命令行执行如下（项目根路径下执行）

BINTRAY_USERNAME 替换 bintray.com 的用户名，BINTRAY_KEY替换复制的 Api key
上传成功后，在刚创建的 maven 厂库中可以看到已上传的这个 package

审核

然后进入这个 package 中,可以看到以后外部引入的代码，compile ‘world.zsp:downloader:1.0’,也就是 compile groupId:artifactId:version 的格式。需要审核后才能使用。现在找到 Add to JCenter 按钮。

之后跳转到发送页面，点 send 按钮，提交发送后，等待 bintray 审核。

通过审核后会收邮件提醒，站内 Message 查看结果. 访问 https://jcenter.bintray.com/world/zsp 可以看到如下结果。world/zsp 为groupId。

总结

准备好 bintray-release 插件所需的 gradle 和 jdk 版本
提交因lint检测失败终止时， 添加 abortOnError 有错误警告时也不会终止打包。最好在 as菜单 -> Analyze -> Inspect Code 逐一排查确认修复。
