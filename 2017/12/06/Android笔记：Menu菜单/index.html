Android笔记：Menu菜单

发表于 2017-12-06 | 分类于 技术

从 Android 3.0（API 级别 11）开始，采用 Android 技术的设备不必再提供一个专用“菜单”按钮。随着这种改变，Android 应用需摆脱对包含 6 个项目的传统菜单面板的依赖，取而代之的是要提供一个应用栏来呈现常见的用户操作。

定义菜单

所有菜单类型，Android 提供了标准的定义格式，在项目 res/menu/ 目录内创建一个 XML 文件。

可以通过以 <item> 子元素的形式添加 <menu> 子菜单

选项菜单

选项菜单是某个 Activity 的主菜单项。放置对应用产生全局影响的操作，如设置、搜索等。选项菜单中的项目在屏幕上的显示位置取决于 Android 版本。

  • Android 2.3.x（API 级别 10）或更低版本，选项菜单的内容会出现在屏幕底部。
  • Android 3.0（API 级别 11）及更高版本，选项菜单中的项目将出现在应用栏中。 默认情况下，系统将所有项目均放入操作溢出菜单中, 要支持快速访问重要操作，将 android:showAsAction=”ifRoom” 添加到对应的 <item>元素，从而将项目提升到应用栏中。

通过 Activity 子类或 Fragment 子类为选项菜单声明项目。如 Activity 和 Fragment 均声明菜单项，则这些菜单项将合并到 UI 中。 系统将首先显示 Activity 的项目，随后按 Fragment 添加到 Activity 中的顺序显示各菜单项。 也可以使用 android:orderInCategory 属性，对需要移动的 <item>重新排序。

创建选项菜单

定义菜单资源文件，然后重写 Activity 或 Fragment 的 onCreateOptionsMenu()

  • Android 2.3.x 及更低版本，用户首次打开时调用 onCreateOptionsMenu。
  • Android 3.0 及更高版本，启动 Activity 时调用 onCreateOptionsMenu。

还可以使用 add() 添加，findItem() 查找，以便修改其 Item 属性。

事件处理

从选项菜单中选择项目（包括应用栏中的操作项目）时，系统将调用 Activity 的 onOptionsItemSelected() 方法。用 getItemId() 识别对应的 Item。

成功处理后，返回 true。未处理，则应调用 onOptionsItemSelected() 的超类实现（默认实现将返回 false）

运行时修改

onCreateOptionsMenu() 仅供创建时使用，如需动态修改菜单项，可通过 onPrepareOptionsMenu() 操作。

  • Android 2.3.x 及更低版本中，每当打开菜单时，调用 onPrepareOptionsMenu()。
  • Android 3.0 及更高版本中，当菜单项显示在应用栏中时，被视为始终处于打开状态。 发生事件时，如果要执行菜单更新，则必须调用 invalidateOptionsMenu() 来请求系统调用 onPrepareOptionsMenu()。

上下文菜单

上下文菜单提供了许多操作，可以为任何视图提供上下文菜单，但这些菜单通常用于 ListView、GridView
菜单操作的方法有两种：

  1. 使用浮动上下文菜单。长按一个声明支持上下文菜单的视图时，菜单弹出显示（类似于对话框）。
  2. 使用上下文操作模式。此模式是 ActionMode 的系统实现，它将在屏幕顶部显示上下文操作栏。


上下文操作模式可用于 Android 3.0（API 级别 11）及更高版本，是显示上下文操作（如果可用）的首选方法。如低于 3.0 版本的系统，则回退到浮动上下文菜单。

创建浮动上下文菜单

1.调用 registerForContextMenu()，注册应菜单关联的 View。如使用 ListView 或 GridView 且需要每个项目提供相同的上下文菜单，通过将 ListView 或 GridView 传递给 registerForContextMenu()，注册所有项目。
2.实现 onCreateContextMenu() 当注册后的视图收到长按事件时，系统将调用 onCreateContextMenu() 方法

3.实现 onContextItemSelected()

使用上下文操作模式

上下文操作模式是 ActionMode 的一种系统实现，它将交互的重点转到执行上下文操作上。通过选择项目启用此模式时，屏幕顶部将出现一个“上下文操作栏”，显示可对当前所选项执行的操作。 当完成或取消操作时，该操作模式将会停用，且上下文操作栏将会消失。

弹出菜单

PopupMenu 是锚定到 View 的模态菜单。如果空间足够，它将显示在定位视图下方，否则显示在其上方。在 API 级别 11 及更高版本中可用。

创建弹出菜单

1.实例化 PopupMenu 及其构造函数，该函数将提取当前应用的 Context 以及菜单应锚定到的 View。
2.使用 MenuInflater 将菜单资源扩充到 PopupMenu.getMenu() 返回的 Menu 对象中。
3.调用 PopupMenu.show()。

弹出菜单事件

实现 PopupMenu.OnMenuItemClickListener 接口，并通过 setOnMenuItemclickListener() 将其注册到 PopupMenu。 选择项目时，系统会调用 onMenuItemClick()。

平台版本不同，支持和显示样式不同。
用符合使用场景需求的菜单
如功能或外观上不满足，自定义菜单解决。
