RxJava总结

发表于 2017-12-14 | 分类于 技术

Reactive Extensions 简写为Rx，是一个编程模型、一种编程思想。目标是提供一致的编程接口，更方便的处理异步数据流。Rx 扩展了观察者模式用于支持数据和事件序列，添加一些操作符，过滤（Filter）、选择（Select）、变换（Transform）、结合（Combine）、组合（Compose）等。可以声明式组合这些序列，无需关注底层实现，高效执行。

名词定义

  • Observable 被观察的对象，数据产生变化时，通知观察者或订阅者。
  • Single 一个特殊的 Observable ，只发射单个数据。
  • Observer 观察者对象，监听 Observable 发射的数据。
  • Subscriber 是 Observer 的一个特殊实现。
  • Subject 同时继承于 Observable 和 Observer。既可以订阅也可以发射。
  • Scheduler 调度器，给操作符增加了多线程功能。

使用步骤

第一步：初始化 Observable
第二步：初始化 Observer
第三步：建立订阅关系

Observable

被观察对象（Observable），观察者对Observable发射的数据做出响应处理。 Observable 模型可以像使用集合一样操作异步事件流，对异步事件流使用各种简单可组合的操作。
访问数据方式类比:

单个数据 多个数据
同步 T getData() Iterable getData()
异步 Future getData() Observable getData()

Observable 既可以像 Future 处理单个数据，更支持处理批量的数据序列。同时拥有 Iterable 一样的功能与灵活性。Observable 是异步双向 push,Iterable 是同步单向 pull。对比：

事件 Iterable(pull) Observable(push)
获取数据 T next() onNext(T)
异常处理 Exception onError(Exception)
任务完成 !hasNext() onCompleted()

可以把 Observable 当做 Iterable 推送方式的等价。

Observer

Observer 即观察者，它决定事件触发的时候将有怎样的行为。 Observable 和 Observer 通过 subscribe() 方法实现订阅关系，从而 Observable 可以在需要的时候发出事件来通知 Observer。
观察者监听的回调方法：

  • onNext():被观察者的把数据流当成一个队列，把每个事件单独处理 触发onNext
  • onCompleted():RxJava 当不会再有新的 onNext() 发出时，触发 onCompleted() 完成。
  • onError(): 事件队列异常。在事件处理过程中出异常时，onError() 会被触发，同时队列自动终止，不允许再有事件发出。
  • 在一个正确运行的事件序列中, onCompleted() 和 onError() 有且只有一个，并且是事件序列中的最后一个。

简单示例

1.Observable.create 方式创建被观察者，OnSubscribe<Integer> 和 new Subscriber<Integer> 的泛型类是对应的
2.当 Observable 被订阅的时候，OnSubscribe 的 call() 方法会自动被调用
3.call 方法中依次调用，订阅者 onNext 的回调方法，完成后调用 onCompleted

Scheduler 线程控制

在不指定线程的情况下，在哪个线程调用 subscribe()，就在哪个线程生产事件并消费事件。如果需要切换线程，就需要用到 Scheduler （调度器）
subscribeOn(): 指定 subscribe() 所发生的线程，即 Observable.OnSubscribe 被激活时所处的线程。或者叫做事件产生的线程。 * observeOn(): 指定 Subscriber 所运行在的线程。或者叫做事件消费的线程。

内置的 Scheduler

  • Schedulers.immediate(): 当前线程运行，默认的 Scheduler。
  • Schedulers.newThread(): 总是启用新线程，并在新线程执行操作。
  • Schedulers.io(): I/O 操作（读写文件、读写数据库、网络信息交互等）所使用的 Scheduler。行为模式和 newThread() 差不多，区别在于 io() 的内部实现是是用一个无数量上限的线程池，可以重用空闲的线程，因此多数情况下 -io() 比 newThread() 更有效率。不要把计算工作放在 io() 中，可以避免创建不必要的线程。
  • Schedulers.computation(): 计算所使用的 Scheduler。这个计算指的是 CPU 密集型计算，即不会被 I/O 等操作限制性能的操作，例如图形的计算。这个 Scheduler 使用的固定的线程池，大小为 CPU 核数。不要把 I/O 操作放在 computation() 中，否则 I/O 操作的等待时间会浪费 CPU。
  • 另外， Android 还有一个专用的 AndroidSchedulers.mainThread()，它指定的操作将在 Android 主线程运行。

observeOn() 可以多次调用，实现了线程的多次切换。

注：subscribeOn() 的位置放在哪里都可以，但它是只能调用一次。如使用了多个 subscribeOn() ，只有第一个 subscribeOn() 起作用。

总结

  • Rx 简化代码、方便处理并发、异步错误处理机制，try/catch没办法处理异步计算。
  • Observable 并不是在创建的时候就立即开始发送事件，而是在它被订阅的时候，即当 subscribe() 方法执行的时候
  • Observable 会持有 Subscriber 的引用，这个引用如果不能及时被释放，将有内存泄露的风险。要在不再使用的时候尽快在合适的地方（例如 onPause() onStop() 等方法中）调用 unsubscribe() 来解除引用关系，以避免内存泄露的发生。
  • 在一个正确运行的事件序列中, onCompleted() 和 onError() 有且只有一个，并且是事件序列中的最后一个。
